# Семантика типов данных языка Pascal++

## Поддерживаемые типы данных

### 1. Числовые типы
| Тип                      | Ключевое слово   | Описание									  | Примеры					|
|--------------------------|----------------  |--------------------------------------------	  |-----------------------	|
|  Число с плавающей точкой  | `float`       |  64-битный тип данных с двойной точностью (double)	| `3.14`, `-2.5`, `0.0`   |

| Тип                      | Ключевое слово   | Описание									  | Примеры					|
|--------------------------|----------------  |--------------------------------------------	  |-----------------------	|
|  Целое число				| `integer`       |  	32-разрядное целое число со знаком		  | `3`, `-2`, `0`   |

### 2. Строковый тип
| Тип    | Ключевое слово | Описание						  | Примеры									  |
|--------|----------------|-----------------------------------|-------------------------------------------|
| Строка | `string`       | Строка символов в кодировке UTF-8 | `"Hello"`, `""`, `"line\n"`				  |


### 3. Массивы

- Тип каждого выражения должен совпадать с типом элемента массива.
- Длина массива определяется количеством при объявлении массива в квадратных скобках.
- Индекс в `a[index]` должен иметь тип `integer`. Выход за границы массива — ошибка времени выполнения.
- Каждый элемент массива — результат вычисления выражения (не обязательно литерал).
- При объявлении массива без явной инициализации полей, они инициализируется по умолчанию в зависимости от типа данных в объявлении структуры и указанной размерности массива

#### Примеры: 
- var a : arr[5] of integer; - объявляется и инициализируется массив [0, 0, 0, 0, 0]
- var b : arr[3] of string - объявляется и инициализируется массив ["", "", ""]
		
### 4. Структуры

- Тип каждого выражения должен совпадать с типом соответствующего поля.
- Доступ к полю: выражение слева от `.` должно иметь тип структуры, содержащей это поле.
- При объявлении структуры без явной инициализации полей, они инициализируется по умолчанию в зависимости от типа данных в объявлении структуры .
- Поля структуры можно инициализировать в любом порядке.

## Типизация выражений

### Арифметические операции:
| Операция    | Типы операндов		 | Тип результата		  |
|-------------|------------------    |------------------------|
| `+ - * / %` | `integer`, `integer` | `integer`			|
| `+ - * / %` | `float`, `float`   | `float`				  |

### Операции со строками:
| Операция    | Типы операндов		 | Тип результата		  |
|-------------|------------------    |------------------------|
| `+`         | `string`, `string`   | `string` (конкатенация)|

### Операции сравнения:
| Операция			| Типы операндов       | Тип результата |
|-------------------|--------------------  |----------------|
| `== != < > <= >=` | `integer`, `integer`   | `integer`	    |
| `== != < > <= >=` | `string`, `string`   | `integer`	    |
| `== != < > <= >=` | `float`, `float`   | `integer`	    |

### Логические операции:
| Операция | Типы операндов      | Тип результата |
|----------|---------------------|----------------|
| `@ \|\| !` | `integer`, `integer` | `integer`	  |

### Операция возведения в степень
| Операция    | Типы операндов		 | Тип результата		  |
|-------------|------------------    |------------------------|
| `^`         | `integer`, `integer`   | `integer`|
| `^`         | `float`, `float`   | `float`|
- 0 ^ x при x < 0 - ошибка времени выполнения (деление на ноль/бесконечность).
- 0 ^ 0 

## Типы функций

### Функции с возвращаемым значением:
```
	function sum(a: integer, b: integer) : integer
	begin
		return a + b;
	end
```

### Функции без возвращаемого значения (void):
```
	function greet(name: string)
	begin
		write("Hello, ", name);
	end
```

### Правила:
1. Функция без `: тип` имеет пустой тип возврата
2. Все ветки выполнения должны возвращать значение совместимого типа
3. Функция с возвращаемым типом должна заканчиваться оператором return выражение
4. Функция без возвращаемого типа заканчивается без return;

### Обработка ошибок типов

	Несоответствие типов при присваивании - ошибка типизации (во время компиляции)

	Неявное приведение типов - ошибка типизации (во время компиляции)

	Отсутствие возврата в функции с типом - ошибка семантики (во время компиляции)

	Возврат несовместимого типа - ошибка типизации (во время компиляции)

	Несовпадение типа индекса с integer при доступе к элементу массива - ошибка типизации (во время компиляции)

	Несовпадение типа элемента в литерале массива с типом массива - ошибка типизации (во время компиляции)

	Обращение к несуществующему полю структуры - ошибка семантики (во время компиляции)

	Не все поля инициализированы в литерале структуры - ошибка семантики (во время компиляции)
	
	Тип значения поля структуры не совпадает - ошибка типизации (во время компиляции)

# Спецификация семантики языка
## Область видимости
- Ограничена блоками кода (`begin...end`)
- Объявленная переменная в области видимости должна использоваться только в этой области видимости и в дочерней области
- После завершения функции, локальная переменная, объявленная в этой области видимости не доступна извне
- Функции не захватывают переменные внешней области видимости

## Уникальность имён
- В пределах одной области видимости не должны повторяться имена переменных, в разных - можно
- Переменные и функции используют разные пространства имён
- Можно объявить переменную с именем, совпадающим с именем функции в другой области видимости
- Нельзя скрыть имена встроенных функций и типов данных
- При наличии переменной с тем же именем во внешней области видимости, внутренняя переменная скрывает внешнюю

## Параметры функций
- Параметры передаются по значению.
- В объявлении функции при указании параметров указывается их тип.
- Тип, количество и порядок параметров в объявлении функции должны совпадать с вызовом функции
- Параметру можно присвоить новое значение внутри функции
- Фактические параметры вычисляются слева направо перед вызовом функции
- Параметр функции может быть скрыт локальной переменной

## Рекурсия
- Внутри исполняемого блока функции может быть вызвана эта же функция
- Поддерживается взаимная рекурсия (две функции могут вызывать друг друга)
- Для взаимной рекурсии функции должны быть объявлены в одной области видимости
- Нет ограничений на глубину рекурсии (ограничено только стеком выполнения)

## Ветвления
- В условиях ветвлений (if) должно использоваться выражение типа integer(0-1)
- При истинном условии выполняется блок после then
- При ложном условии выполняется блок после else (если присутствует)
- Нет аналога switch..case
- Все блоки в ветвлениях должны быть явными (begin...end)

## Циклы
### Цикл while
- Выполняется, пока выражение в условии возвращает true
- Условие должно иметь тип integer(0-1)
- Выражение условия вычисляется перед каждой итерацией
- Если условие ложно с самого начала, цикл не выполняется ни разу

### Цикл for
- Идентификатор после for создаёт новую переменную в области видимости цикла
- Выражения from и to вычисляются один раз перед началом цикла
- Переменная цикла изменяется от начального значения до конечного включительно
- На каждой итерации проверяется, не превысила ли переменная цикла конечное значение
- Тип переменной цикла и границ должен быть integer

#### Операторы управления циклами
- break - немедленно завершает выполнение текущего цикла
- continue - переходит к следующей итерации текущего цикла
- break и continue допустимы только внутри циклов  

## Оператор return
- Может использоваться только в блоках функций

## Присваивания
- В левой части присваивания допускаются: идентификатор переменной, доступ к элементу массива `a[index]`, доступ к полю структуры `obj.field`
- Нельзя присваивать литералам, вызовам функций или произвольным выражениям
- Тип присваемого выражения должен соответствовать объявленному типу переменной
- Тип переменной нельзя изменить после объявления

## Инициализация переменных
- При объявлении переменной без явной инициализации, она инициализируется по умолчанию (string - "", integer - "0", float - "0.0" ).
- Параметры функций считаются инициализированными
- Переменная цикла for инициализируется значением выражения from

## Оператор read

- Для переменных типа integer: ожидается ввод числа в десятичной записи без точки, допускается ведущий знак +/-
- Для переменных типа float: ожидается ввод дробного числа в десятичной записи с точкой, допускается ведущий знак +/-
- Для переменных типа string: ожидается ввод строки в двойных кавычках ("...")
- ведущие/хвостовые пробелы разрешены: они игнорируются.

## Оператор write

Семантика оператора write:

1. Для выражений типа float: если число имеет нулевую дробную часть — может выводиться без дробной части” (или наоборот: всегда с точкой).

2. Для выражений типа string: выводится строка без кавычек write("Hello") - выведет: Hello

3. Символы экранирования (\n, \t) при выводе интерпретируются как реальные символы

4. Для массивов:
	- Нельзя передать в write весь массив(ошибка типизации), только элемент массива
	- Каждый элемент выводится по правилам своего типа

5. Для структур
	- можно выводить только значение поля структуры write(person.name)
	- запрещено печатать значения структур целиком - ошибка типизации

6. Несколько выражений выводятся подряд, без разделителей
   - write("x=", x); при x=5 выведет: x=5
   - Для пробелов нужно явно указывать: write("x = ", x);

7. write() всегда содержит перевод строки в конце

8. Ведущие/хвостовые пробелы в выражении вне кавычек разрешены - они игнорируются.
